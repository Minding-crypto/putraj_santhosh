Speech Impediment Learning
Speech Enhancement · Python · DSP · Deep Learning

Worked in the audio R&D team on a hard real-world problem: restoring speech quality in environments with unpredictable, non-stationary noise. Built a Python pipeline that fused classical Digital Signal Processing with deep learning models, targeting inference latency tight enough for real-time use.

Veridion
AI decision forensics · Mechanistic interpretability · Bias monitoring

An AI governance platform that turns open-weight model decisions into signed, mechanistic evidence — showing what information mattered inside the model, where it was represented, and whether changing those internal signals changes the outcome. Targeted at regulated teams in banking, HR tech, insurance, and healthcare who need more than policy checklists.

The core is a local runner that audits a focused prompt family without requiring weight uploads or raw regulated data to leave the customer environment. For each decision, Veridion scores a logit margin, runs activation patching (circuit sufficiency and necessity tests), checks minimality by pruning unneeded nodes, and applies a bias-pressure screen that flags cases where the visible answer stays unchanged but the decision margin shifts against a protected proxy after FDR correction. A certificate is only issued after all gates pass — sufficiency, necessity, minimality, negative controls, and random-circuit controls. If a circuit cannot be proven, the system returns "review" or "not certifiable" instead of fabricating a mechanistic story. Output is a signed JSON/PDF artifact bundle consumed by compliance, engineering, and sales teams.

Waste Audit Console
Computer vision · Waste classification · Sustainability reporting

A browser-based audit console that lets operators scan physical waste items with a camera, classify them in real time using an AI vision model, and generate signed PDF compliance reports — all without any server-side upload of raw images.

Images are captured via the device camera or file upload, base64-encoded in the browser, and sent to a vision model configured with a fail-closed ensemble policy — items with non-empty containers, visible contamination, sharp hazards, or special-disposal signals are excluded from the recyclable stream regardless of model confidence. Classification results are written to a live audit record that tracks material type, decision, confidence score, estimated weight, and CO2e avoided. A filter and search layer over the Audit Records table lets reviewers drill into any decision. The dashboard aggregates recycling rate, recyclable weight, fail-closed rate, average confidence, false-recyclable rate, and image quality score. Report generation renders an executive-summary PDF with claim boundaries, material breakdown, and audit method disclosure directly from a print template — no backend required.

Vision model classification Fail-closed ensemble policy Real-time audit trail CO2e tracking PDF report generation Browser-native · no backend
DevOps Code Quality Portal
Full-stack · Angular · Spring Boot · PostgreSQL · SonarQube

A multi-tenant DevOps platform that lets engineering teams run automated code quality scans against any GitHub repository branch and track quality trends over time. Teams define per-project quality gate policies, manage members with role-based access, and view a live dashboard of bugs, vulnerabilities, coverage, and code smells.

Built on a three-tier architecture: an Angular standalone-component frontend, a Spring Boot backend using a layered controller-service-repository pattern with Spring Data JPA, and PostgreSQL with JSONB columns for schema-flexible metric storage. Scan execution is fully asynchronous — the trigger endpoint returns a PENDING scan record immediately while a Spring @Async thread clones the repo via Git, invokes SonarQube as a subprocess, polls for completion, evaluates results against the project's policy thresholds, and persists the final run. Auth is stateless JWT via Spring Security with HMAC-SHA256 signing, enabling horizontal scaling with no session store required. Trend charts are rendered as raw SVG in the browser with no charting library — data is normalised client-side and delta KPIs computed directly from scan history.

BlocBox
Cross-border logistics · React Native · Stripe

Singaporeans can browse and buy from Malaysian e-commerce platforms, but getting the goods across the border is the hard part. BlocBox solves the last-mile problem with a proprietary locker network and a mobile app that handles reservations, payments, and multi-stage delivery tracking — all in one flow.

Automotive OSINT Pipeline
AI threat intelligence · Gemini · Llama

An autonomous engine that continuously scrapes the web for automotive cybersecurity vulnerabilities, filters noise, and produces structured compliance-aligned reports — running without human input.

Raw articles and forum posts are ingested from a set of automotive security sources, then passed through MinHash locality-sensitive hashing to deduplicate near-identical content before it reaches the LLM stage — cutting downstream processing cost significantly. A dual-LLM gate uses Gemini for broad semantic relevance classification and Llama for precise CVE and regulatory entity extraction, with each model's output cross-validated to reduce false positives. Final reports are structured around ISO 21434 threat categories and auto-tagged with CVE identifiers for consumption by security teams.

Gaia's Hand
FinTech / disaster relief · ILP · ClickHouse · Twilio

Disaster aid suffers from two problems: money moves slowly and aid can't reach people without internet. Gaia's Hand automates the full lifecycle — pooling donations, running a supplier bidding process, and distributing aid to recipients who only have a basic phone.

Donation pooling and supplier payouts run over the Interledger Protocol, enabling micro-payment settlement across currency boundaries without a central intermediary. Supplier reliability scores are computed in ClickHouse using a rolling window over historical delivery fulfilment data, and the bidding engine weights bids by both price and score. Aid distribution uses Twilio to send SMS messages containing encoded redemption tokens; recipients reply with the token to trigger QR voucher generation, which local partner agents verify offline — the system works end-to-end with no smartphone or data connectivity required on the recipient's side.
